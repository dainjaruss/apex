APEX production UI directions

Capstone is done — these mockups show a denser, operations-first shell with light and dark themes. Review in the browser before any Next.js implementation.

Screens

  • 01 — Dashboard / evaluation queue
  • 02 — Evaluation workspace (section nav + validation)
  • 03 — Sign in

Design intent

  • Table-first queue with filters (status, routing stage, reporting period)
  • Persistent global search and clear routing-stage chips
  • Three-column edit: sections · form · validation & routing summary
  • Sticky save bar — always visible primary actions
  • Sidebar stays navy in both themes for brand continuity
